# 多手机号统一管理APP - 测试指南

## 项目概述
这是一个集成三大运营商（中国移动、中国联通、中国电信）的多手机号统一管理平台MVP版本。

## 技术栈
- 前端框架：Vue 3 + uni-app
- 状态管理：Pinia
- 样式：SCSS
- 语言：TypeScript

## 环境配置
### API配置文件
位置：`src/config/index.ts`

**配置说明：**
```typescript
// 环境切换：DEV / TEST / PROD
const currentEnv = 'DEV'

// 运营商API
OPERATOR_API_CONFIG = {
  cmcc: { /* 中国移动配置 */ },
  cucc: { /* 中国联通配置 */ },
  ctcc: { /* 中国电信配置 */ }
}

// 支付配置：微信支付 / 支付宝
PAYMENT_CONFIG = { /* ... */ }
```

### Mock模式开关
所有Store中都有 `USE_MOCK = true` 配置，设置为 `false` 即可接入真实API。

## API接口定义

### 认证接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/auth/send-code` | 发送验证码 |
| POST | `/auth/login` | 登录 |
| POST | `/auth/logout` | 登出 |
| POST | `/auth/refresh` | 刷新Token |
| POST | `/auth/verify-realname` | 实名认证 |
| POST | `/auth/change-password` | 修改密码 |

### 号码接口
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/numbers/list` | 获取号码列表 |
| POST | `/numbers/add` | 添加号码 |
| GET | `/numbers/detail/:id` | 号码详情 |
| PUT | `/numbers/update` | 更新号码 |
| POST | `/numbers/delete` | 删除号码 |
| POST | `/numbers/freeze` | 冻结号码 |
| POST | `/numbers/unfreeze` | 解冻号码 |
| POST | `/numbers/:id/sync` | 同步号码数据 |

### 运营商接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/operator/verify` | 验证号码归属人 |
| POST | `/operator/info` | 获取号码信息 |
| POST | `/operator/balance` | 查询余额 |
| POST | `/operator/bills` | 查询账单 |
| POST | `/operator/recharge` | 充值 |
| POST | `/operator/invoices/available` | 可开票账单 |
| POST | `/operator/invoices/create` | 开发票 |

### 缴费接口
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/payment/create` | 创建订单 |
| POST | `/payment/params` | 获取支付参数 |
| GET | `/payment/query/:orderNo` | 查询订单 |
| GET | `/payment/records` | 充值记录 |
| POST | `/payment/retry` | 重试支付 |

### 发票接口
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/invoice/list` | 发票列表 |
| GET | `/invoice/detail/:id` | 发票详情 |
| GET | `/invoice/available` | 可开票账单 |
| POST | `/invoice/create` | 开具发票 |
| GET | `/invoice/download/:id` | 下载发票 |
| POST | `/invoice/forward/:id` | 转发发票 |

## 测试步骤

### 快速开始测试
1. 启动开发服务器：`npm run dev:h5`
2. 访问：http://localhost:5173/
3. 登录页面输入任意手机号（如：13812345678）
4. 验证码：任意6位数字
5. 密码：Test1234（8-16位含字母数字）
6. 勾选协议后登录

### 完整测试流程

#### 测试用户系统
1. 验证登录页面UI和交互
2. 测试各输入框的表单验证
3. 登录成功后的跳转
4. 刷新页面验证登录状态保持
5. 退出登录功能

#### 测试号码管理
1. 在号码页面点击添加号码
2. 填写测试信息（如：13900001111，中国移动）
3. 验证号码列表展示
4. 测试排序功能
5. 点击进入号码详情
6. 编辑备注、设为常用
7. 测试冻结/解冻
8. 删除号码

#### 测试缴费
1. 进入缴费页面
2. 选择一个号码
3. 选择充值金额
4. 模拟支付
5. 查看充值记录

#### 测试发票
1. 进入发票页面
2. 选择一个可开票账单
3. 填写发票信息
4. 提交开具

#### 测试安全中心
1. 查看安全日志
2. 测试号码冻结

## 数据安全
### 数据加密
- 敏感数据传输：AES-256
- 手机号展示：中间4位脱敏（138****5678）
- 身份证号：脱敏展示
- 本地Token加密存储

### 合规要求
- 所有操作跳转官方渠道
- 不存储运营商密码
- 用户授权后才能访问数据
- 符合数据安全法规
